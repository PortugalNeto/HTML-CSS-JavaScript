     b. Sobre o pensamento computacional

Após ter lido alguns artigos eu diria que o pensamento computacional é, nada mais, nada menos, que o conjunto intersecção da maneira que nós pensamos com a maneira que o computador “entenda”, a união de tudo o que está envolto nesta intersecção é, ao meu ver, pensamento computacional.

A seguir explico sucintamente o que seriam os 4 vetores do pensamento computacional que, quando somados, são capazes de resolver qualquer tipo de problema/tarefa, das mais simples às mais complexas:

  1. Decomposição

    2. Encarar um problema grande como o somatório de pequenos problemas, de maneira que estes problemas menores sejam de mais fácil resolução;

  2. Reconhecimento de Padrões

    3. Com o tempo em resolução de problemas dos mais variados tipos, é provável que se encontre padrões em determinados tipos de resoluções que podem ser aplicados a outros problemas;

  3. Abstração

    4. Esta é a “peça fundamental”, onde não basta fazer a leitura do problema mas enxergar quais pontos são essenciais e serão utilizados na composição da “receita” e quais não têm relevância ou esta é menor;

  4. Criação do Algoritmo

    5. Unindo os três pontos supracitados somos capazes de criar uma série de definições e regras a serem seguidas em uma sequência exata que o computador entenda e executa a fim de atingir nossos objetivos. Sim, tudo é possível ao que crê (e ao que "coda").


Questão 2-

     a. O front-end de uma aplicação, como um site por exemplo, é tudo que pode ser diretamente visualizado e acessado pelo usuário. O código (ou hypertexto) do HTML é renderizado pelo navegador, formando exatamente esta página que estamos observando. O CSS é a estilização da página, como cores degradê, botões que se modificam ao "passar o mouse sobre", abas laterais, uma infinidade de coisas que agregam ao visual final. Por fim, a linguagem de programação mais comumente utilizada no Front-end é o JavaScript, cujo objetivo é fazer todo o algoritmo que faça o botão '+' de uma calculadora, por exemplo, executar a soma de dois algarismos recebidos como parâmetros de uma função de JavaScript.
      Já o back-end é "o que não vai ser renderizado", é do "clique" que gera o protocolo de requisito pra "baixo". Quando uma rede social me pede meu login e senha por um formulário de texto, após eu digitar e enviar essa informação isto vai ser enviado para o Para back-end utilizamos Node.js (linguagem JavaScript), Java, C#, Python, .Net, Cobol e muitas outras linguagens. O exemplo clássico de front-end e back-end é o restaurante onde, o lugar que você senta, tudo o que você vê, o menu para os pedidos, tudo isso pode ser considerado front-end. Quando o garçom vem e anota o seu pedido é feita uma request, ela pode estar disponível (status 200) ou indisponível (status 404, ele pode dizer que aquele prato está indisponível). Se o pedido estiver disponível, o garçom levará à cozinha (back-end) para a preparação do que foi pedido. Entre o Front e o Back há o conceito de API que é uma sigla para "Application Programming Interface" que é quem lida com as requisições (como Get, put, delete e outras) e no restaurante seria o análogo ao garçom que anota seu pedido e leva à cozinha.

     b. Os bancos de dados podem ser considerados "repositórios" de dados, neles unimos os dados e através dos Sistemas de Gerenciamento de Banco de Dados (ou SGBD) manipulamos os dados de um banco. Eles podem ser considerados bancos de dados relacionais (como o Oracle que é o mais utilizado, ou o MySQL ou o SQL Server dentre muitos outros) ou NoSQL (Not Only SQL, não apenas relacional) como o MongoDB, Cassandra e uma infinidade além.
